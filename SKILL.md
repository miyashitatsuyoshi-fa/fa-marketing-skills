# イベントLP SEO・CRO監査スキル

このスキルは、BtoBイベントやセミナーのランディングページ（LP）を分析し、検索エンジンからの自然流入（SEO）と、訪問者の申し込み率（CRO）を最大化するための具体的な改善提案レポートを作成するためのものです。

## 監査プロセス

以下のステップに従って監査を実施し、レポートを作成してください。

### 1. 対象LPのクロールと技術的SEO分析
対象となるLPのURLにアクセスし、以下の技術的要素を確認します。
- **メタタグ:** `title`, `meta description`, `h1`〜`h6`タグの有無と内容の適切さ。
- **構造化データ:** `application/ld+json`（特に`Event`スキーマ）が実装されているか。
- **パフォーマンス:** ページの読み込み速度（TTFBなど）、リソース（JS/CSS）の読み込み状況。
- **OGPタグ:** SNSシェア時の表示設定（`og:title`, `og:description`, `og:image`など）。

### 2. 競合分析とキーワード戦略調査
イベントのテーマ（例：「新リース会計基準」）に関連するキーワードで検索を行い、以下の点を調査します。
- **検索市場の動向:** ターゲット層がどのようなキーワードで検索しているか（情報収集、実務対応、ソリューション探索など）。
- **競合他社の状況:** 競合がどのようなセミナーを開催し、どのような訴求を行っているか。
- **キーワード戦略の提案:** LPに盛り込むべきターゲットキーワードとロングテールキーワードの提案。

### 3. UX/UI・CRO評価
LPのユーザー体験（UX）とコンバージョン最適化（CRO）の観点から、以下の要素を評価します。
- **ファーストビュー:** 開催日時、場所、登壇者、CTA（Call to Action）が明確に配置されているか。
- **コンテンツ構成と信頼性:** ターゲットの課題に刺さるコピー、データ、参加者の声などの信頼性要素が適切に配置されているか。
- **フォームとCTA:** 申し込みフォームの使いやすさ（iframeの有無など）、CTAボタンの視認性と配置。

### 4. 包括的レポートの作成
上記の分析結果を統合し、以下の構成でMarkdown形式のレポートを作成します。

#### レポート構成案
1. **エグゼクティブサマリー:** 監査結果の全体的な評価と主要な改善ポイント。
2. **技術的SEO分析:** メタタグ、構造化データ、パフォーマンスの評価と改善提案。**必ず実装可能なコードスニペット（HTML/JSON-LD）を含めること。**
3. **競合分析とキーワード戦略:** 検索動向、競合状況、ターゲットキーワードの提案。
4. **UX/UIとコンバージョン最適化（CRO）評価:** ファーストビュー、コンテンツ構成、フォーム/CTAの評価と改善提案。**必要に応じてCSSなどのコードスニペットを含めること。**
5. **優先順位付けされた改善提案と工数見積もり:** 施策内容、期待される効果、想定工数、担当領域をまとめた表。
6. **結論と今後のステップ:** 最優先で取り組むべき事項と、中長期的な視点でのアドバイス。

## 重要な注意事項
- **具体的なコードスニペットの提供:** 改善提案には、ユーザーがそのままコピー＆ペーストして実装できる具体的なコード（例：`<meta name="description" content="...">` や `Event` スキーマのJSON-LD）を必ず含めてください。
- **優先順位の明確化:** 改善提案は、インパクト（効果）とエフォート（工数）を考慮し、優先順位（高・中・低）をつけて提示してください。
- **懐疑的な視点からのリスク検証:** レポートの最後、またはユーザーへの報告メッセージ内で、提案した施策が期待通りの効果を生まないリスク（例：イベントまでの期間が短くSEO効果が限定的である可能性など）についても言及し、客観的な視点を提供してください。
- **ブランドガイドラインの遵守:** ファーストアカウンティング株式会社などの特定の企業のLPを監査する場合は、その企業のブランドカラーやトーン＆マナーを意識した提案（例：CTAボタンの色指定など）を行ってください。
